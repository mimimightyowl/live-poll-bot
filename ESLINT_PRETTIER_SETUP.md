# ESLint и Prettier настройка

## Установленные пакеты

- `eslint` - линтер для JavaScript/TypeScript
- `prettier` - форматтер кода
- `eslint-config-prettier` - отключает правила ESLint, конфликтующие с Prettier
- `eslint-plugin-prettier` - интегрирует Prettier в ESLint
- `@eslint/js` - базовая конфигурация ESLint
- `husky` - Git хуки для автоматизации
- `lint-staged` - запуск линтеров только на staged файлах

## Доступные команды

### Линтинг

```bash
npm run lint          # Проверить код на ошибки
npm run lint:fix      # Исправить автоматически исправимые ошибки
```

### Форматирование

```bash
npm run format        # Форматировать весь код
npm run format:check  # Проверить форматирование без изменений
```

### Pre-commit хуки

```bash
# Автоматически запускается при git commit
# Форматирует и проверяет только staged файлы
git commit -m "your message"
```

## Конфигурация

### ESLint (eslint.config.js)

- Использует новый формат конфигурации ESLint v9
- Базовые правила из `@eslint/js`
- Интеграция с Prettier
- **Все правила настроены как предупреждения** (не блокируют коммиты)
- Предупреждения для `console.log`, неиспользуемых переменных и других проблем

### Prettier (.prettierrc)

- Одинарные кавычки
- Точки с запятой
- Ширина строки: 80 символов
- Отступы: 2 пробела
- Unix окончания строк (LF)

### Lint-staged (package.json)

- Автоматически форматирует staged файлы с помощью Prettier
- Запускает ESLint с автоисправлением на staged JS/TS файлах
- Форматирует JSON, MD, YAML файлы
- Работает только с файлами, которые вы добавляете в коммит

### Игнорируемые файлы

- `.prettierignore` - файлы, которые не форматируются
- `.gitignore` - обновлен для игнорирования кэша ESLint/Prettier

## Текущие предупреждения

В коде есть несколько предупреждений:

- Использование `console.log` (можно заменить на логгер)
- Неиспользуемая переменная `pool` в server.js

**Важно:** Эти предупреждения НЕ блокируют коммиты благодаря настроенной конфигурации ESLint. Код будет автоматически форматироваться при коммите, но предупреждения останутся для информации.
